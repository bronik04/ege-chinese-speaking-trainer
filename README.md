# Тренажёр устной части ЕГЭ по китайскому языку

Локальный тренажёр по вариантам ЕГЭ 2021–2026. В нём есть полный экзаменационный режим, отдельная тренировка заданий 1–3, официальные тайминги, запись с микрофона, история прогресса, учебные группы и проверка назначенных работ преподавателем.

## Запуск

Из папки проекта выполните:

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/uvicorn asgi:app --host 127.0.0.1 --port 8080
```

Затем откройте `http://127.0.0.1:8080`. Для записи разрешите браузеру использовать микрофон.

`server.py` сохранён как совместимый переходный сервер, но для разработки и публикации используется FastAPI + Uvicorn через `asgi.py`.

## Публичное развёртывание

```bash
cp .env.example .env
# укажите DOMAIN, SMTP и при необходимости R2/OpenAI
docker compose up -d --build
```

Контейнер приложения работает без root на Uvicorn. Caddy принимает порты 80/443, автоматически получает и продлевает TLS-сертификат, включает сжатие и проксирует запросы в приложение. DNS домена должен указывать на сервер, а порты 80 и 443 должны быть открыты. Альтернативная заготовка для nginx находится в `deploy/nginx.conf`; сертификаты для неё нужно настроить отдельно.

Размер запроса ограничен на прокси и в ASGI-приложении. Аудио дополнительно проверяется через `ffprobe`: допустимый максимум — 30 секунд для задания 1, 150 секунд для задания 2 и 210 секунд для задания 3.

Обычный Docker Compose использует SQLite и сохраняет прежнюю простоту запуска. PostgreSQL не обязателен и подключается только при росте нагрузки:

```bash
docker compose -f compose.yml -f compose.scale.yml up -d --build
```

В scale-профиле PostgreSQL 16 использует пул соединений, а схема обновляется Alembic. Та же версия клиента `pg_dump` зафиксирована в Docker и CI. SQLite продолжает использовать собственные версионированные миграции и остаётся полностью поддерживаемым основным режимом.

Резервная копия PostgreSQL создаётся через `pg_dump`; в локальном SQLite-режиме используется SQLite Backup API. Локальные аудиофайлы архивируются отдельно:

```bash
docker compose exec app python scripts/backup.py --data-dir /app/var --output-dir /app/backups --keep 14
```

Команду следует запускать ежедневно через cron/systemd timer и копировать каталог `backups/` на другой сервер или в объектное хранилище. Для R2 настройте отдельное резервное копирование бакета у провайдера: скрипт не скачивает весь объектный архив на сервер. Периодически проверяйте восстановление копии, а не только факт её создания.

Переключатель «Быстрый режим» сокращает таймеры и удобен для демонстрации. Записи свободных тренировок хранятся только в памяти текущей вкладки; записи назначенных преподавателем работ загружаются на сервер.

## Прогресс и аккаунты

- Без входа история и настройки сохраняются в `localStorage` текущего браузера.
- После регистрации или входа прогресс синхронизируется с SQLite-базой `var/trainer.sqlite3` либо с опциональным PostgreSQL.
- При регистрации можно выбрать роль ученика или преподавателя.
- Преподаватель создаёт учебные группы с шестизначным кодом и видит статистику подключившихся учеников.
- Ученик вводит код в личном кабинете, после чего его прогресс становится доступен преподавателю этой группы.
- Преподаватель назначает группе вариант и состав заданий; каждая сдача сохраняется как отдельная попытка.
- Аудиозаписи сохраняются локально или в S3/Cloudflare R2 и доступны через API только ученику и преподавателю группы.
- Отдельный воркер автоматически расшифровывает китайскую речь; статус и текст видны преподавателю рядом с плеером.
- Проверка использует критерии ФИПИ: максимум 5 баллов за задание 1, 7 за задание 2 и 8 за задание 3.
- Пароли хранятся только в виде PBKDF2-хешей с индивидуальной солью.
- Сессия передаётся в `HttpOnly` cookie со сроком 30 дней.
- Email подтверждается одноразовой ссылкой со сроком 24 часа.
- Ссылка восстановления пароля действует один час и завершает все прежние сессии после смены пароля.
- Ограничения попыток входа, регистрации и восстановления сохраняются в БД и переживают перезапуск сервера.
- В личном кабинете доступен журнал входов и важных учебных действий; аккаунт можно удалить вместе с прогрессом и аудиозаписями.

Папка `var/` исключена из Git и не должна публиковаться. Схема обновляется при запуске. Для публичного размещения запускайте сервер за HTTPS-прокси и установите `TRAINER_SECURE_COOKIE=1`.

## Ошибки и наблюдаемость

Все ошибки API имеют единый машинно-читаемый формат:

```json
{
  "code": "invalid_credentials",
  "message": "Неверный email или пароль",
  "requestId": "c5f1d43f8cf94a488afe9ab85d891d25"
}
```

`X-Request-ID` возвращается в каждом FastAPI-ответе и связывает ответ с JSON-логом сервера. Логи содержат метод, путь, статус и длительность запроса; необработанные исключения фиксируются с traceback. `/api/health` публикует только агрегированные счётчики ответов 4xx/5xx и время последней серверной ошибки, без пользовательских данных. Уровень логирования задаётся через `TRAINER_LOG_LEVEL`.

## PostgreSQL, R2 и расшифровка

PostgreSQL нужен только для масштабирования. Запустите `compose.scale.yml` или задайте стандартный URI подключения:

```bash
DATABASE_URL=postgresql://trainer:password@postgres:5432/trainer
```

Для Cloudflare R2 создайте закрытый бакет и токен только с правами чтения/записи этого бакета, затем задайте:

```bash
TRAINER_AUDIO_STORAGE=s3
TRAINER_S3_BUCKET=chinese-ege-audio
TRAINER_S3_ENDPOINT_URL=https://ACCOUNT_ID.r2.cloudflarestorage.com
AWS_ACCESS_KEY_ID=...
AWS_SECRET_ACCESS_KEY=...
AWS_DEFAULT_REGION=auto
```

Для автоматической расшифровки включите воркер и передайте ключ только через секреты окружения:

```bash
TRAINER_TRANSCRIPTION_ENABLED=1
OPENAI_API_KEY=...
OPENAI_TRANSCRIPTION_MODEL=gpt-4o-mini-transcribe
TRAINER_TRANSCRIPTION_LANGUAGE=zh
```

Загрузка ответа не ждёт внешний API: запись попадает в устойчивую очередь БД. Воркер повторяет временные ошибки до трёх раз и возвращает зависшее задание в очередь через 15 минут. Запуск отдельно от Compose: `.venv/bin/python -m scripts.transcription_worker`.

В Docker воркер включается отдельным профилем: `docker compose --profile transcription up -d --build`. Его можно сочетать как с основным SQLite-режимом, так и с `compose.scale.yml`.

## Почта и ссылки аккаунта

Без SMTP письма не отправляются наружу: локальные сообщения записываются в закрытый файл `var/outbox.log`. Это удобно для разработки — последняя строка содержит ссылку подтверждения или восстановления.

Для публичного запуска задайте переменные окружения:

```bash
export TRAINER_PUBLIC_URL="https://trainer.example.ru"
export TRAINER_SMTP_HOST="smtp.example.ru"
export TRAINER_SMTP_PORT="465"
export TRAINER_SMTP_SSL="1"
export TRAINER_SMTP_USER="smtp-user"
export TRAINER_SMTP_PASSWORD="smtp-password"
export TRAINER_SMTP_FROM="noreply@example.ru"
```

Для SMTP на порту 587 используйте `TRAINER_SMTP_SSL=0` и `TRAINER_SMTP_STARTTLS=1`. Пароли и SMTP-настройки нельзя добавлять в репозиторий — передавайте их через секреты окружения хостинга.

## Проверка

Основные тесты авторизации, миграций, загрузки аудио, проверки работ и структуры вариантов запускаются без дополнительных зависимостей:

```bash
python3 -m unittest discover -s tests -v
```

Полная локальная проверка, совпадающая с GitHub Actions:

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements-dev.txt
npm install
.venv/bin/pre-commit install
.venv/bin/ruff check .
npm run lint:js
npm test
npx playwright install chromium
npm run test:e2e
.venv/bin/coverage run -m unittest discover -s tests -v
.venv/bin/coverage report
```

Перед коммитом можно проверить весь репозиторий одной командой: `.venv/bin/pre-commit run --all-files`. Она запускает Ruff, ESLint и проверку синтаксиса JSON. CI использует ту же конфигурацию, запускает модульные и Playwright E2E-тесты, проверяет PostgreSQL 16 с Alembic и восстановлением `pg_dump`, а S3-контракт — через MinIO. Покрытие Python-кода должно быть не ниже 55%.

## Структура кода

- `asgi.py` и `api/routes/` — нативные маршруты FastAPI;
- `api/schemas.py` — Pydantic-контракты входных данных и схема OpenAPI;
- `api/controllers/` — контроллеры аккаунтов, групп, работ и записей;
- `server.py` — компактный совместимый локальный сервер;
- `backend/` — база данных, миграции, запросы, безопасность и оценивание;
- `app.js` — точка сборки экранов и общего состояния;
- `js/account-controller.js` — координатор кабинета, а тематические `account-*-controller.js` отвечают за авторизацию, безопасность, группы, назначения и проверку;
- `js/runner-controller.js` — экзаменационный сценарий;
- `migrations/` — Alembic-миграции опционального PostgreSQL;
- `tests/`, `tests-js/` и `tests-e2e/` — API-, клиентские и браузерные тесты.

## Добавление вариантов

Каждый вариант хранится отдельно в `data/variants/`. Чтобы добавить новый, создайте JSON по структуре существующих файлов и зарегистрируйте его в `data/variants/index.json`. Изображения вариантов находятся в `assets/variants/<год>/`, используют WebP и единый шаблон имени `candidate-XX.webp`. Автотест контролирует наличие файлов и общий бюджет изображений до 3 МБ.
