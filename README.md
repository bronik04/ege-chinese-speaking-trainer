# Тренажёр устной части ЕГЭ по китайскому языку

Локальный тренажёр по вариантам ЕГЭ 2021–2026. В нём есть полный экзаменационный режим, отдельная тренировка заданий 1–3, официальные тайминги, запись с микрофона, история прогресса, учебные группы и проверка назначенных работ преподавателем.

## Запуск

Из папки проекта выполните:

```bash
python3 server.py
```

Затем откройте `http://127.0.0.1:8080`. Для записи разрешите браузеру использовать микрофон.

Переключатель «Быстрый режим» сокращает таймеры и удобен для демонстрации. Записи хранятся только в памяти текущей вкладки и исчезают после её закрытия.

## Прогресс и аккаунты

- Без входа история и настройки сохраняются в `localStorage` текущего браузера.
- После регистрации или входа прогресс синхронизируется с SQLite-базой `var/trainer.sqlite3`.
- При регистрации можно выбрать роль ученика или преподавателя.
- Преподаватель создаёт учебные группы с шестизначным кодом и видит статистику подключившихся учеников.
- Ученик вводит код в личном кабинете, после чего его прогресс становится доступен преподавателю этой группы.
- Преподаватель назначает группе вариант и состав заданий; каждая сдача сохраняется как отдельная попытка.
- Аудиозаписи назначенных работ сохраняются в `var/audio/` и доступны только ученику и преподавателю группы.
- Проверка использует критерии ФИПИ: максимум 5 баллов за задание 1, 7 за задание 2 и 8 за задание 3.
- Пароли хранятся только в виде PBKDF2-хешей с индивидуальной солью.
- Сессия передаётся в `HttpOnly` cookie со сроком 30 дней.

Папка `var/` исключена из Git и не должна публиковаться. Схема SQLite обновляется версионированными миграциями при запуске. Для публичного размещения запускайте сервер за HTTPS-прокси, установите `TRAINER_SECURE_COOKIE=1` и перенесите аудио в объектное хранилище.

## Проверка

Основные тесты авторизации, миграций, загрузки аудио, проверки работ и структуры вариантов запускаются без дополнительных зависимостей:

```bash
python3 -m unittest discover -s tests -v
```

Полная локальная проверка, совпадающая с GitHub Actions:

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements-dev.txt
npm install
.venv/bin/ruff check .
npm run lint:js
npm test
.venv/bin/coverage run -m unittest discover -s tests -v
.venv/bin/coverage report
```

Dev-инструменты не нужны для запуска приложения: сервер и интерфейс по-прежнему работают без сторонних runtime-зависимостей. CI проверяет Python через Ruff, JavaScript через ESLint, запускает Python- и JavaScript-тесты и требует покрытие Python-кода не ниже 55%.

## Структура кода

- `server.py` — HTTP-маршрутизация и обработчики API;
- `backend/` — база данных, миграции, запросы, безопасность и оценивание;
- `app.js` — управление состоянием интерфейса и экзаменационным сценарием;
- `js/` — API-клиент, прогресс, шаблоны кабинета, проверка работ и отображение заданий;
- `tests/` и `tests-js/` — серверные, интеграционные и клиентские тесты.

## Добавление вариантов

Каждый вариант хранится отдельно в `data/variants/`. Чтобы добавить новый, создайте JSON по структуре существующих файлов и зарегистрируйте его в `data/variants/index.json`. Изображения вариантов находятся в `assets/variants/<год>/`, используют WebP и единый шаблон имени `candidate-XX.webp`. Автотест контролирует наличие файлов и общий бюджет изображений до 3 МБ.
