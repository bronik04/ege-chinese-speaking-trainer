# ADR 0001: FastAPI как основной runtime

**Статус:** Accepted

## Контекст

В репозитории одновременно существуют FastAPI-приложение в `asgi.py` и совместимый HTTP-сервер в `server.py`. Две production-точки входа увеличивают объём проверок и риск расхождения HTTP-контрактов.

## Решение

FastAPI через `asgi.py` является единственным целевым production runtime. Новые маршруты, middleware и HTTP-возможности реализуются только в FastAPI.

`server.py` получает статус legacy и сохраняется временно для совместимости существующих сценариев. Его перенос или удаление выполняется отдельной задачей после проверки, что runtime, тесты, Docker, worker и документация от него не зависят.

## Последствия

- Публичный HTTP-контракт развивается в одном framework.
- Docker, deployment и новые тесты ориентируются на Uvicorn/FastAPI.
- До удаления legacy часть общих сценариев может проверяться дважды.
- Исправления безопасности, применимые к legacy до его удаления, всё ещё должны переноситься в `server.py`.

## Ограничения

- Это решение не разрешает немедленно удалить `server.py`.
- Текущие URL и форматы ответов сохраняются без отдельного API-решения.
- `asgi.py` остаётся тонким compatibility entrypoint для приложения из `src/trainer/main.py`.
