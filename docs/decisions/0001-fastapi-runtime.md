# ADR 0001: FastAPI как основной runtime

**Статус:** Accepted

## Контекст

В репозитории исторически существовали FastAPI-приложение в `asgi.py` и совместимый HTTP-сервер в корне. Две production-точки входа увеличивали объём проверок и риск расхождения HTTP-контрактов.

## Решение

FastAPI через `asgi.py` является единственным целевым production runtime. Новые маршруты, middleware и HTTP-возможности реализуются только в FastAPI.

Compatibility server изолирован в `legacy/server.py`. Основные тесты, Docker, worker и CI от него не зависят; его ручная проверка сохранена до завершения одного стабильного релизного цикла.

## Последствия

- Публичный HTTP-контракт развивается в одном framework.
- Docker, deployment и новые тесты ориентируются на Uvicorn/FastAPI.
- Legacy-сценарии проверяются только ручно и не влияют на основной coverage gate.
- Исправления безопасности, применимые к compatibility runtime до его удаления, всё ещё должны переноситься в `legacy/server.py`.

## Ограничения

- Каталог `legacy/` удаляется только после одного стабильного релизного цикла.
- Текущие URL и форматы ответов сохраняются без отдельного API-решения.
- `asgi.py` остаётся тонким compatibility entrypoint для приложения из `src/trainer/main.py`.
